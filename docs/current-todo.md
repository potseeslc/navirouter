# Current TODO

This is the cleaned-up todo list after comparing the original NaviRouter plan with the current repo state.

## What We Already Proved

- [x] NaviRouter stays a router, not a replacement music player.
- [x] Normal Navidrome/Subsonic traffic works through one NaviRouter URL.
- [x] AudioMuse powers radio/similar-song responses when a client calls compatible endpoints.
- [x] Arpeggi browse, streaming, library switching, radio, and queue matching are verified.
- [x] Library switching is preserved through Navidrome's normal `musicFolderId` behavior.
- [x] Unscoped radio can infer the library from the seed track when `NAVIDROME_DB_PATH` is configured.
- [x] AudioMuse queues can be saved, generated by API, or synced from an external AudioMuse workflow as normal Navidrome playlists.
- [x] The web dashboard exists as a control room for health, source attribution, recent calls, queue proof, and diagnostics.
- [x] The repo is ready for public GitHub sharing with Docker, security, release, smoke-test, and community-focused docs.
